Miniature Automated Car
(Real-Time System)

Project Summary

During my Real-Time System course I was tasked with designing and creating an automated vehicle which would be able to handle user input as well as be able to partially drive automatically. To do this I was given a raspberry Pi and a robot car which consisted of wheels, a distance and line sensor, motors, a camera, and built in horn. Although the hardware was provided the car had no previously implemented functionality. Using a virtual machine running Ubuntu and the Eclipse IDE I implemented the code and then cross compiled and uploaded it to the Raspberry Pi which was hooked onto the car. By the end of the quarter I created a robot car that could take user input to drive in any direction from a separate machine which sent input via a TCP connection as well as receive back a video which was sent as a series of UDP packets from the robot car. Along with this the car was able to follow a black line without steering off course using its built in line sensors and automatically stop and begin beeping if it was about to hit a wall. The system was optimized using multithreading and rate monotonic scheduling to ensure that all tasks were meeting their deadlines and appropriate amount of CPU was being used.

Project Analysis

This project allowed me to learn a lot about how real time systems worked and how to effectively use multithreading and scheduling to handle multiple tasks at once. Along with that I learned much about working in embedded systems where you need to handle different physical and digital inputs and outputs. This project required an in-depth understanding in both the hardware and software aspects to tackle issues such as determining stopping distance and adjusting motor speeds to take turns.
