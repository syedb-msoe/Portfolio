SchedTool

Project Summary

As part of MSOE's software development laboratory course I was put into a group with 4 others to take an ongoing project known as SchedTool. SchedTool was a tool designed to be used by the school registrar to help them find conflicts when planning courses for the next term. SchedTool would take in an in-progress course plan as well as a year and major and display the courses based on the plan, major, and year. Registrar could then move courses around to find a plan that worked and export the course plan. We were given a year to work on this project and finish it up but managed to complete it in around 3 months.

Project Analysis

This project was a good learning experience since most of the projects I had done up until this one had been started from scratch. This project however was a continuation from the past year which meant that a lot of the technologies were already selected for us and there was a lot of code that we hadn't written and needed to understand. I ended up learning a lot about certain tools and js and python frameworks such as yarn, flask, react.js, and FullCalendar. The project was difficult at first since we were not accustomed to the code or technologies. After we managed to understand it however, we were able to make a lot of improvements and get the project to the point where the PO felt comfortable calling the project done fairly quickly. 

The multiview tool allowed registrar to compare different course timings and see if there were any room conflicts

The calendar view displays all available courses for a specific major, year, and term. The registrar can use this data to make sure there aren't any courses that would conflict with each other
