Modem Activated Warning System

Project Summary

For my senior design project I teamed up with a group of 4 other people and TAPCO to develop a prototype for a modem activated warning system for their traffic signs. Currently TAPCO's street signs relied on radio signals to communicate with one another which struggled over long distances and going through obstacles such as trees. An example of this was one of their 'Road May Flood' signs which would flash an led if a flood was detected ahead. This required the sign and sensor to be put far away from each other which was difficult with radio waves. We were constrained to using less than 100mW of power and the connection needed to occur over cellular data.

Project Analysis

This project is still a work in progress so this analysis will be updated as we go farther into the process. There seems to be a lot of new skills that will need to be picked up on the project based on our initial analysis. Most of my knowledge is software oriented but there are a lot of hardware components related to this project such as the power constraint and the fact that we need to determine the parts required and put them together ourselves. The project will also require a large knowledge on network protocols since we will have to design one to allow the signs to speak to one another. It will be expected that we will use C or assembly for this project which I have some experience on  but I will most likely need to go even deeper on those topics as well
